# SEO

## Decisions

### Strategy

The site is a personal portfolio. SEO serves two purposes:
1. **Discoverability**: Rank for "Hugo Bollon" and related name queries.
2. **Credibility**: When someone googles the name (recruiters, colleagues), the site should appear professional and complete.

This is not a content marketing site. There is no blog, no organic traffic strategy. SEO is about correctness and completeness, not competitive keyword ranking.

### Rendering and indexability

- SSG produces fully rendered HTML. Every page is immediately indexable without JavaScript execution.
- No client-side rendering issues for search crawlers.
- Google, Bing, and social media crawlers receive complete HTML with all content and meta tags.

### Meta tags

Managed by `composables/useSeo.ts`, called in every page.

#### Required meta tags per page

| Tag                    | Source                          | Fallback                                        |
|------------------------|---------------------------------|-------------------------------------------------|
| `<title>`              | Strapi SEO component `metaTitle`| `Hugo Bollon — DevOps Engineer & Fullstack Dev` |
| `meta[description]`    | Strapi SEO component `metaDescription` | Static fallback string                   |
| `meta[viewport]`       | Static                          | `width=device-width, initial-scale=1`           |
| `meta[charset]`        | Static                          | `utf-8`                                         |
| `link[canonical]`      | Computed from route + site URL  | —                                               |
| `html[lang]`           | Current i18n locale             | `en`                                            |

#### Open Graph tags

| Tag              | Source                             |
|------------------|------------------------------------|
| `og:title`       | Same as `<title>`                  |
| `og:description` | Same as `meta[description]`        |
| `og:image`       | Strapi SEO `ogImage` or default    |
| `og:url`         | Canonical URL                      |
| `og:type`        | `website`                          |
| `og:locale`      | `en_US` or `fr_FR` based on i18n  |

#### Twitter Card tags

| Tag                  | Value                         |
|----------------------|-------------------------------|
| `twitter:card`       | `summary_large_image`         |
| `twitter:title`      | Same as `og:title`            |
| `twitter:description`| Same as `og:description`      |
| `twitter:image`      | Same as `og:image`            |

### Default Open Graph image

- Dimensions: 1200x630px.
- Stored at `public/og-image.jpg`.
- Contains: name, title, visual brand elements.
- Can be overridden per-page via Strapi SEO component.

### i18n SEO

#### Hreflang tags

Generated automatically by `@nuxtjs/i18n` when `strategy: 'prefix_except_default'` is used.

Expected output:
```html
<link rel="alternate" hreflang="en" href="https://hugobollon.dev/" />
<link rel="alternate" hreflang="fr" href="https://hugobollon.dev/fr/" />
<link rel="alternate" hreflang="x-default" href="https://hugobollon.dev/" />
```

#### Locale-specific URLs

| Locale | URL pattern        |
|--------|--------------------|
| EN     | `https://hugobollon.dev/` (no prefix, default) |
| FR     | `https://hugobollon.dev/fr/`                    |

### Sitemap

Generated by `@nuxtjs/sitemap` module.

Configuration:
- Hostname: `NUXT_PUBLIC_SITE_URL` environment variable.
- Auto-discovers routes from `app/pages/` directory (Nuxt 4 srcDir).
- i18n alternates included automatically.
- Output: `public/sitemap.xml` during `nuxt generate`.

### robots.txt

Static file at `public/robots.txt`:
```
User-agent: *
Allow: /

Sitemap: ${NUXT_PUBLIC_SITE_URL}/sitemap.xml
```

Note: The robots.txt file uses the canonical domain `hugobollon.dev`. Example: `https://hugobollon.dev/sitemap.xml`.

### Structured data (JSON-LD)

Decided as "optional" in session. Recommended for v1 but not blocking:

```json
{
  "@context": "https://schema.org",
  "@type": "Person",
  "name": "Hugo Bollon",
  "jobTitle": "DevOps Engineer",
  "url": "https://hugobollon.dev",
  "sameAs": [
    "https://github.com/hbollon",
    "https://linkedin.com/in/hugo-bollon"
  ]
}
```

Implementation: `<script type="application/ld+json">` in the `<head>`, generated by `useSeo()` composable or directly in `app.vue`.

### Favicon

- `public/favicon.ico` (multi-size ICO or single 32x32).
- Optionally: `apple-touch-icon.png` (180x180), `favicon-32x32.png`, `favicon-16x16.png`.
- `site.webmanifest` for PWA metadata (optional, not a priority).

## Assumptions

- The canonical site URL is **`https://hugobollon.dev`**. This is set in the `NUXT_PUBLIC_SITE_URL` environment variable.
- `@nuxtjs/i18n` correctly generates hreflang tags for SSG output. This must be verified after implementation.
- Google Analytics is not relevant for SEO ranking but is mentioned here for completeness: it must be loaded with GDPR consent to avoid legal risk, not for SEO reasons.

## Open questions

1. ~~**Canonical domain**~~: **Resolved. `hugobollon.dev`.** All other domains should 301 redirect to the canonical one.
2. **Structured data scope**: `Person` schema only, or also `WebSite`, `CreativeWork` (for projects)? Additional schemas improve rich results but add implementation effort.
3. **Google Search Console**: Should be set up after deployment to monitor indexation. Not an implementation concern but a post-launch action item.

## Risks

- **Duplicate content between /en/ and /**: If `prefix_except_default` is misconfigured, both `/` and `/en/` could serve the same content without canonical tags. `@nuxtjs/i18n` handles this but must be verified.
- **OG image not found**: If Strapi SEO component has no `ogImage` and the default `public/og-image.jpg` is missing, social previews will be broken. Ensure default image always exists.
